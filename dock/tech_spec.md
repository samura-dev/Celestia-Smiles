# 📐 Техническая Архитектура и Стек: Zub Clinik (Portfolio Edition)

Данный документ описывает техническое устройство, стек и архитектурные решения для разработки премиального сайта стоматологической клиники.
**Цель проекта:** Пет-проект под портфолио/Awwwards с максимальным упором на визуал, 3D, WebGL и smooth-скроллинг. SEO не в приоритете.

---

## 🛠 1. Глобальный Технический Стек

Поскольку SEO не критично, мы уходим от тяжеловесных SSR-решений (Next.js) в сторону максимально быстрого и гибкого Client-Side Rendering (CSR), что развязывает нам руки для работы с канвасом и скролл-анимациями.

### Ядро (Core)

* **Фреймворк:** React 18 + Vite (Максимальная скорость HMR и сборки).
* **Язык:** TypeScript (Строгая типизация всех пропсов, стейтов и API-ответов).
* **Роутинг:** React Router DOM v6 (Глобальный `BrowserRouter` или `RouterProvider` для SPA навигации без перезагрузок).
* **Стейт-менеджер:** Zustand (Легковесный глобальный стейт, идеально подходит для шаринга данных между обычным React-деревом и Three.js `Canvas`).

### UI & Стилизация

* **Стилизация базы:** Tailwind CSS (Утилиты) + Vanilla CSS/CSS Modules для специфичных сложных шейпов и градиентов.
* **Дизайн система:** Никаких Material UI / Chakra. Все компоненты (`sd_button`, `sd_card`) пишем с нуля руками ради чистоты кода и премиальности. Используем `clsx` + `tailwind-merge` для удобной конкатенации классов.
* **Стиль:** Glassmorphism, полупрозрачности (blur), гладкие монохромные градиенты, глубокий тёмный режим.

### Анимации & WebGL

* **3D База:** Three.js + React Three Fiber (`@react-three/fiber`) + Drei (`@react-three/drei`).
* **Плавный скролл:** Lenis (от Studio Freight). Это стандарт для Awwwards-решений.
* **Таймлайны и Скролл-триггеры:** GSAP + ScrollTrigger (Анимации контента по скроллу, параллаксы, пин элементов).
* **Интерфейсные анимации:** Framer Motion (Для плавного появления модалок, аккордеонов FAQ, page-transitions, и эффектов курсора).

## 🏗 2. Архитектура Проекта (FSD-lite)

Структура директорий `src` будет разделена по смысловым блокам:

```text
src/
├── assets/         # Шрифты, иконки, текстуры для Three.js
├── components/     # UI Kit приложения
│   ├── ui/         # Базовые атомы (кнопки, инпуты, аккордеоны)
│   └── layout/     # Header, Footer, CustomCursor
├── gl/             # ВСЁ что касается WebGL и 3D (React Three Fiber)
│   ├── models/     # Компненты 3D моделей (напр. Стеклянный Зуб)
│   ├── shaders/    # GLSL файлы для слайдера До/После и искажений
│   └── scenes/     # Сборки 3D сцен для разных блоков
├── hooks/          # Кастомные хуки (напр. useSdMousePosition, useSdSmoothScroll)
├── pages/          # Страницы (Home, Services, Doctors)
├── store/          # Zustand сторы
├── styles/         # Глобальные CSS файлы и Tailwind директивы
└── utils/          # Хелперы, функции математики (напр. lerp)
```

## ⚙️ 3. Инженерные решения под сложные фичи

### 3.1. Глобальный стеклянный 3D Зуб (Hero Section -> Manifesto)

**Проблема:** Зуб появляется на главном экране и должен реагировать на скролл, уплывая за границы при переходе ко второму блоку, не перегружая DOM.
**Решение:** Единственный глобальный компонент `<Canvas>` фиксируется под всем приложением (`position: fixed; inset: 0; z-index: -1`, `pointer-events: none`). Все 3D-сцены рендерятся там. Связь скролла и 3D координат организуется хуком `useScroll` из `drei` или кастомной привязкой позиции скролла к положению камеры/модели через GSAP. Для преломления света юзаем `MeshTransmissionMaterial`.

### 3.2. WebGL Слайдер «До / После»

**Проблема:** Классический `<input type="range">` со сдвигом `clip-path` картинки - это скучно и не тянет на премию.
**Решение:** Пишем кастомный фрагментный шейдер (GLSL). Загружаем 2 текстуры (фото "до" и фото "после"). Привязываем хук движения мыши/свайпа к `uniform float uProgress;` шейдера. В шейдере математически искажаем UV-координаты текстур при `uProgress` от 0 до 1, создавая эффект "жидкого" или глитч-перехода.

### 3.3. Темизация на лету (Блок с оборудованием)

**Проблема:** При доскролливании до определенного блока сайт должен резко, но плавно стать тёмным (sci-fi стиль).
**Решение:**

1. Создаем `useSdThemeStore` в Zustand: `{ theme: 'light' | 'dark' }`.
2. В компоненте `Блок 6: Оборудование` ставим GSAP ScrollTrigger, который при пересечении триггера экрана вызывает `setTheme('dark')` в глобальный стор, при уходе — `setTheme('light')`.
3. Корневой элемент `<div id="root">` подписан на этот стор и переключает класс `dark` для Tailwind. Свойство `transition-colors duration-700` обеспечит плавность.

### 3.4. Page Transitions (Переходы между страницами SPA)

**Решение:** Используем `<AnimatePresence mode="wait">` из Framer Motion вокруг `<Outlet />` React Router'a. Это позволит нам проигрывать красивые анимации (например, наезд черной шторки) перед тем, как компонент текущей страницы исчезнет и отрендерится новая.

## 📜 4. Code Style & Правила Написания

(Основано на правилах проекта)

1. **Именование:** `camelCase` для функций/переменных, `PascalCase` для компонентов.
2. **Префикс:** Все функции и классы обязательно должны иметь префикс `sd_` (пример: `sd_container`, `const sd_handleScroll = () => {}`).
3. **Отступы:** 2 пробела, обязательное использование точки с запятой `;`.
4. **Рефакторинг:** Жесткое разделение бизнес-логики и UI. Максимальный вынос стейтов в Zustand и хуки (`useSd*`). Компоненты страниц должны быть тонкими, описывающими только рендер дочерних блоков.
5. **Комментарии:** Весь код, особенно математика в хуках и GLSL шейдеры, должен обильно комментироваться на русском языке.
